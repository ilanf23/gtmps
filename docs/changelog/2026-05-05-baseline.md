## Rollback guidance

If a new change regresses behavior, revert the specific PR after this baseline rather than trying to reconstruct older history.

For urgent restoration:

1. Use Lovable History for the fastest full project restore.
2. Use GitHub PR history to locate the smallest safe revert.
3. Check affected routes, content modules, migrations, and backend functions listed in the PR.

## Going forward

Every meaningful change should leave three artifacts:

- a focused branch
- a reviewed PR
- a changelog entry when the product surface materially changes
