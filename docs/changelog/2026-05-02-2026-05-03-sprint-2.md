## 4. Vertical landings — template-driven refactor

The eight vertical pages (`/consulting`, `/law`, `/accounting`, `/msp`, `/advisory`, `/ae`, `/recruiting`, `/agency`) were refactored away from the monolithic `VerticalLanding` component.

### New structure
```
src/pages/verticals/
├── _template/
│   ├── VerticalPage.tsx     ← single page renderer (~900 LOC)
│   ├── configs.ts           ← per-vertical config
│   ├── data.tsx             ← per-vertical content (~1.1k LOC)
│   └── css.ts               ← per-vertical CSS-in-JS
├── Consulting.tsx           ← <VerticalPage config={consulting} />
├── Law.tsx                  ← <VerticalPage config={law} />
└── ... (8 total)
```

Each vertical page is now a one-liner:
```tsx
import VerticalPage from "./_template/VerticalPage";
import { consulting } from "./_template/data";
export default function ConsultingPage() {
  return <VerticalPage config={consulting} />;
}
```

### Removed
- `src/components/VerticalLanding/VerticalLanding.tsx` (897 LOC)
- `src/components/VerticalLanding/VerticalNav.tsx` (299 LOC)
- `src/components/VerticalLanding/VerticalFooter.tsx` (202 LOC)
- `src/components/VerticalLanding/VerticalStickyCta.tsx`

### Added
- **`src/components/VerticalLanding/VerticalNavBar.tsx`** (551 LOC) — new vertical nav with a firm menu (cross-vertical switcher).

### What `src/content/verticals.ts` is now
Still the source of truth for the **vertical metadata** (slug, nav links, short labels, types). Consumed by `Discover`, `About`, `VerticalNavBar`, and the cohort lib. **Page content has moved to `src/pages/verticals/_template/data.tsx`.** Editing copy on a vertical page now means editing `_template/data.tsx`, not `content/verticals.ts`.

> **Doc update needed elsewhere:** `CLAUDE.md` and `docs/02-architecture.md` both still say "edit `src/content/verticals.ts`" for vertical copy. That is now half-true (metadata yes, page content no). Architecture doc has been updated; CLAUDE.md should be updated next session if/when the project chooses one canonical home for vertical content again.
